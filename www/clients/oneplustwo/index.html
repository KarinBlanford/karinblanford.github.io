Responsive vs Adaptive Web Design

Web pages and the the information on them should be easily legible and navigable across devices including mobile phones, tablets, and desktop PC’s.

This means that the components, visual elements, and text blocks that render in an organized fashion on a large monitor must also have a natural flow and fit when a user accesses a webpage on a small mobile device.

There are two common approaches to designing and building these views: responsive and adaptive.

Responsive web design lays out elements across a page (a header, body, footer, photos, etc) in such a way that as the user’s screen size shrinks or expands, the elements themselves shrink and expand according to the page width. Elements that once spanned across the page left to right might cascade down vertically on smaller form factor devices.

Adaptive web design techniques are used when shrinking or expanding a component or moving it beneath another still causes issues for the user when viewing on mobile devices. For example, a complex navigation menu with multiple elements might be challenging for a user to view and use on a smaller device (i.e. an iPhone 5). Using an adaptive approach, on smaller devices, the user would see a different version of the navigation menu but optimized for the smaller view.

In general, a responsive approach is preferred across web pages, however, there are use cases where an adaptive technique provides a better experience for the user. A combination of approaches can be used within the same web page.
